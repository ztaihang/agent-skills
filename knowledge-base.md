## 2026-06-17-01 标题“AI味”过重
**现象**：封面主标题与徽标用语偏“模型化”，观感不亲和。  
**影响**：降低平台原生感与分享意愿。  
**处理**：标题改为“实战派技能真香榜”等编辑化表达；徽标词汇改为“必装技能”。  
**预防**：标题用“价值导向+场景化+可执行”三段法；避免“AI 顶流”等夸张词。

## 2026-06-17-02 背景色不符平台气质
**现象**：初版背景偏冷或粉色过饱和。  
**处理**：新增并启用 `xhs-warm` 主题（暖白纸感+低饱和珊瑚橙点缀）。  
**预防**：面向小红书发布时，优先暖白/低饱和、弱阴影；统一一条强调色。

## 2026-06-17-03 徽标表情显“机器感”
**现象**：封面徽标前缀表情导致“AI味”。  
**处理**：移除表情，仅保留文本；徽标可置空不渲染。  
**预防**：平台内容默认禁用表情装饰，除非是品牌语气明确要求。

## 2026-06-17-04 单卡信息密度失衡
**现象**：先后尝试添加“加分点/安装提示/要点卡片”等，显拥挤；又出现“内容偏少”的反复。  
**处理**：最终定为“标题 + 标签 + 1 段描述 + 场景”四要素；删除“安装提示”“三个列表”。  
**预防**：信息密度控制在“一屏核⼼卖点”；平台长文说明放到帖文正文，不塞入图片。

## 2026-06-17-05 单卡垂直对齐不优
**现象**：主体内容偏上，空白区过大。  
**处理**：卡片容器引入 `card-main`，使用上下 `auto` 外边距实现垂直居中；封面不受影响。  
**预防**：静态海报类画布固定 1080×1440，内容模版先做居中版，再按需求上下微调 20–40px。

## 2026-06-17-06 构建报错：content.json 结构不匹配
**现象**：build 报“缺少 title 或 items”。  
**原因**：模板期望 `title/subtitle/intro` 为顶层字段，而非 header 嵌套。  
**处理**：将字段改为扁平结构，`items[*].scenes` 统一为字符串。  
**预防**：遵循 `references/content-schema.md`，新建内容先跑一次 `npm run build` 快速验证。

## 2026-06-17-07 导出失败：Playwright 下载中断（ECONNRESET）
**现象**：`npx playwright install chromium` 多次失败导致无法导出。  
**处理**：修改 `scripts/export.mjs`，导出时优先尝试本机 Chrome/Edge（`channel: "chrome" | "msedge"`），并支持 `CHROME_PATH`/`PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH` 环境变量；仅在完全找不到浏览器时再提示下载。  
**预防**：网络不稳场景优先走“本机浏览器回退”；镜像源命令写入导出脚本错误提示。

## 2026-06-17-08 出图规格确认
**规范**：画布固定 1080×1440（3:4），DPR=2；禁止动画；统一主题；单卡上下居中。  
**预防**：导出前检查预览页 `preview/*.html` 与边距；导出后抽查 cover 与 card-01。

## 2026-06-17-09 分页与要点样式争议
**现象**：页码“X/Y”与要点卡片造成视觉噪音。  
**处理**：页面移除页码；要点样式由“卡片/左边线/圆点”逐级收敛到“无”。  
**预防**：默认不开启页码与要点模块；如需加信息密度，采用正文增加 1 行“最佳用法举例”替代。

# 统一作业清单（Quick Checklist）
1. 内容校对：标题编辑化、描述≤50字、场景短语化  
2. 主题选择：小红书发布→`xhs-warm`；避免高饱和、重阴影、表情  
3. 构建验证：`npm run build`，无 schema 报错后再预览  
4. 垂直对齐：单卡启用居中容器，必要时上下微调 20–40px  
5. 导出策略：优先本机 Chrome/Edge → 环境变量 → Playwright 下载（镜像备选）  
6. 质检抽样：封面 + card-01 + 随机一张，检查对齐/对比/文本可读性
